ProgressPrinter.byte_progress_bar module¶

class ProgressPrinter.byte_progress_bar.ByteProgressBar(byte_size, conversion=<UNITS.AUTO: 5>, length=50, pre='', post='', fill='=', head='>', empty=' ')¶

Bases: ProgressPrinter.progress_bar.ProgressBar

This class extends the base ProgressBar module and adds unit conversion capabilities for working with bytes.

Initializes a ByteProgressBar object.

Parameters

  • byte_size – The length of the progress to show in bytes

  • conversion – The unit to convert the bytes to for display. Omit to automatically infer the best unit from the byte size. For supported values see the UNITS enum.

  • length – The length of the progress bar (really… I’m only talking about the bar). Defaults to 50.

  • pre – A string that is printed before the initial empty progress bar is shown. Omitted by default.

  • post – A string that is printed after the progress bar finished. Omitted by default.

  • fill – The string that is used to fill the body of the advancing progress bar. Defaults to ‘=’.

  • head – A string that if not emtpy is displayed as the head of the advancing progress bar. Defaults to ‘>’.

  • empty – A string that is used to fill the empty space that is not filled by the progress bar. Defaults to spaces.

print_progress(current_byte=0, pre='')¶

Method to start printing the previously set-up progress bar with the converted values. Call without current_byte and pre before starting whatever you want to track with the progress bar to initialize the progress bar to the terminal (show an emtpy progress bar).

Parameters

  • current_byte – The current byte progress to display in the bar. Defaults to 0.

  • pre – A string that is appended to the console output above the progress bar. Useful to display additional information like file names. Omitted by default.
